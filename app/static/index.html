Sample data is a synthetic hospital dataset (no real PHI). The dirty variant has intentional defects so you can see every check in action.

 

One table per file. Foreign-key checks need parent tables, so they are skipped for single-file uploads — use the sample audit to see them run.

 

Resources/segments are mapped to the table model, given surrogate integer keys, and audited together — cross-table integrity included. Try the bundled sample_data/fhir/sample_bundle.json or sample_data/hl7/sample_adt.hl7.

 

Every check is recorded — summary only, never patient data. Run a few checks, then watch a dataset's quality score trend over time.
